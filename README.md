# Depth vs Width in MLPs: What Matters More for Classification?

**A Machine Learning Tutorial by Md Minhajul Hayat Mim (24129906)**

## Overview

This tutorial systematically compares **deep-narrow** versus **wide-shallow** Multilayer Perceptron architectures on a handwritten digit classification task. By holding parameter counts roughly equal, we isolate the effect of architecture shape on learning speed, generalisation, and error patterns.

## Key Findings

- Balanced architectures (2–3 layers, 48–64 neurons) tend to perform best
- Very deep, narrow networks (6 layers × 16 neurons) underperform due to bottleneck effects
- Wide-shallow networks are fast to train but may miss compositional features
- Parameter efficiency varies significantly across architecture shapes

## How to Run

```bash
git clone https://github.com/minhajul-mim/mlp-depth-vs-width.git
cd mlp-depth-vs-width
pip install -r requirements.txt
jupyter notebook notebook/mlp_depth_vs_width_tutorial.ipynb
```

Run all cells sequentially. Figures are saved to `figures/` automatically. The dataset is loaded from scikit-learn (no download needed).

## Dataset

**Scikit-learn Digits** — 1,797 images of handwritten digits (0–9), 8×8 pixels each (64 features). Built into scikit-learn.
